DAY 03 – GIT, GITHUB, AGILE & SCRUM NOTES

GIT & VERSION CONTROL
Git is a distributed version control system that tracks code changes and maintains project history. 
It enables safe collaboration, rollback to previous versions, and better team coordination.

Setup:
git config --global user.name
git config --global user.email

Core Commands:
git clone <url>      - copy remote repository
git status           - check file state
git add .            - stage changes
git commit -m "msg"  - save snapshot
git pull origin main - fetch updates
git push origin main - upload changes

File States:
Modified -> Staged -> Committed

Command Line Basics:
ls/dir to list files, cd to change folders, bash to run commands.

GITHUB
GitHub hosts repositories online for sharing, backup, and teamwork.
Supports branches, pull requests, issue tracking, and collaboration.

AGILE
Iterative development approach delivering small, frequent improvements.
Focuses on flexibility, quick feedback, and customer value.

Agile Values:
• Individuals over processes
• Working software over documentation
• Collaboration over contracts
• Responding to change

SCRUM
A framework to implement Agile practices using short cycles called sprints.
Includes sprint planning, daily standups, reviews, retrospectives, story points, and velocity tracking.

OUTCOME
Gained hands-on experience with Git workflow, repository management, GitHub collaboration, and practical understanding of Agile and Scrum for structured and efficient software development.
